### 1.2 通用規範

- 協議：HTTPS
- API 格式：RESTful
- 內容類型：JSON (application/json)
- 字元編碼：UTF-8
- API URL 格式：[protocol]://[host]/[version]/[resource-path]
  > 例如：https://api.ai-coach.ai-liaise.com/[ API路徑]

### 1.3 命名規範

- URI 路徑使用小寫英文字母
- 資源名稱使用名詞形式
- 查詢參數使用駝峰式命名
- JSON 屬性名稱使用駝峰式命名

### 1.4 日期時間格式

- 日期時間格式遵循 ISO 8601 標準
- 日期時間：YYYY-MM-DDThh:mm:ss.sssZ (例如：2024-07-23T22:26:54.474772)
- 純日期：YYYY-MM-DD (例如：2024-11-22)

## 2. 認證與授權

### 2.1 認證方式

API 使用 Bearer Token 認證機制，客戶端在每次請求時需在 HTTP Header 中加入授權資訊：

```
Authorization: Bearer {access_token}
```
