## 1. B2C 電商平台：標準購物網站

### 場景說明
適用於建立面向一般消費者的線上購物平台，涵蓋從商品瀏覽到訂單完成的完整流程。

### Prompt 模板
```
# 專案：B2C 電商購物平台需求分析

## 輸入材料
- 首頁截圖：顯示商品分類、熱銷商品、搜尋欄
- 商品列表頁截圖：包含篩選條件（價格、品牌）、排序選項
- 商品詳情頁截圖：有商品圖片、規格選擇（尺寸、顏色）、"加入購物車"按鈕
- 購物車頁面截圖：列出已選商品、可調整數量、顯示總金額、有"前往結帳"按鈕

## 核心需求
1. 會員系統：支援用戶註冊、登入、查看訂單歷史
2. 金流整合：需要整合信用卡、Apple Pay、Google Pay 等至少三種支付方式
3. 物流追蹤：用戶能在訂單頁面查看即時物流狀態
4. 後台管理：管理員需要一個後台來管理商品、訂單和客戶資料

## 關鍵 KPI
- 頁面載入時間需低於 2 秒
- 需支援至少 1,000 人同時在線瀏覽

請執行完整的 AISDLC 流程，產出包含前後端規格的完整需求文件。
```

## 2. 企業管理：內部 ERP 系統升級

### 場景說明
針對希望將老舊內部管理系統（如進銷存）現代化的企業，著重於資料遷移、效能提升與行動化支援。

### Prompt 模板
```
# 專案：ERP 系統現代化升級

## 現況與問題
我們公司使用一套超過 10 年的桌面端進銷存系統，存在以下問題：
1. 技術老舊：系統頻繁崩潰，維護困難
2. 操作複雜：界面不直觀，新員工上手至少需要一週
3. 資料孤島：無法與新的財務軟體整合，需要人工重複輸入資料
4. 缺乏行動支援：銷售人員在外無法即時查詢庫存和下單

## 升級需求
1. 核心功能保留：需完整保留現有的商品、採購、銷售、庫存管理功能
2. 技術架構：升級為 B/S 架構的 Web 應用，並支援 RWD
3. 資料遷移：現有超過 50 萬筆歷史資料需無痛遷移至新系統
4. 效能提升：查詢複雜報表的回應時間必須在 5 秒內
5. API 整合：提供標準 API 接口，用於與外部財務系統對接

請分析此系統升級的需求與挑戰，並規劃一個分階段實施的建議方案。
```

## 3. 行動 APP：健身與健康追蹤

### 場景說明
適用於開發一款整合穿戴式裝置數據、提供個人化運動計畫與社群互動功能的健康管理 App。

### Prompt 模板
```
# 專案：AI 健身追蹤 APP

## 核心概念
打造一款智慧健身夥伴 App，幫助用戶記錄運動、追蹤健康數據，並透過社群激勵保持運動習慣。

## 主要功能
1. 運動記錄：
   - 支援多種運動類型（跑步、重訓、瑜珈）
   - 透過 GPS 記錄戶外運動軌跡與數據
2. 健康數據整合：
   - 整合 Apple HealthKit 與 Google Fit 數據，同步體重、心率、睡眠資料
   - 提供健康數據的趨勢圖表分析
3. 社群互動：
   - 用戶可分享運動成果到動態牆
   - 建立運動挑戰活動，邀請朋友參加
4. 個人化計畫：
   - 根據用戶設定的目標（減重、增肌）推薦每週運動計畫
   - 設立成就獎章系統，激勵用戶達成目標

## 技術要求
- 支援 iOS 和 Android 雙平台
- 需要離線功能，在無網路狀態下也能記錄運動

請分析此健身 APP 的詳細需求，特別是使用者體驗和後端資料處理部分。
```

## 4. 金融科技：行動銀行 App

### 場景說明
適用於開發安全的行動銀行應用，重點在於交易安全、法規遵循以及與銀行核心系統的整合。

### Prompt 模板
```
# 專案：行動銀行 App 核心功能開發

## 專案類型：行動金融服務應用

## 輸入材料
- 設計稿截圖 1 (首頁)：顯示帳戶總覽、餘額、快速功能入口（轉帳、付款）
- 設計稿截圖 2 (轉帳頁)：包含收款人選擇、金額輸入、轉帳說明欄位

## 核心需求
1. 帳戶管理：使用者能查詢帳戶餘額、最近六個月的交易明細
2. 轉帳功能：支援銀行帳號轉帳，並可儲存常用收款人
3. 安全機制：
   - 登入需支援生物辨識（指紋/臉部）
   - 所有超過 1,000 元的交易皆需透過簡訊 OTP 進行二次驗證
4. 法規遵循：系統設計需符合金融監管機構的資訊安全規範與個資保護法

## 技術要求
- 確保 99.99% 的系統可用性
- 所有敏感資料在傳輸與儲存時皆須全程加密

請執行完整的 AISDLC 流程，產出符合金融級安全標準的系統規格文件 (SRD)。
```

## 5. 教育科技：線上學習平台

### 場景說明
適用於建立企業內訓或公開課程的線上教育平台，涵蓋課程管理、學員互動與學習成效分析。

### Prompt 模板
```
# 專案：企業線上學習平台 (LMS)

## 目標
為企業打造一個提供員工技能培訓與知識管理的線上學習平台。

## 主要功能
1. 課程管理：
   - 管理員可上傳多媒體課程內容（影片、PDF、簡報）
   - 可規劃系列課程的「學習路徑」
2. 學員功能：
   - 學員可觀看課程影片、記錄筆記、追蹤學習進度
   - 完成課程後可進行線上測驗，並在通過後自動頒發電子證書
3. 互動設計：
   - 每個課程皆有專屬討論區，供學員與講師問答
4. 數據分析：
   - 管理者可查看學員完課率、測驗成績等學習成效報表

## 技術要求
- 影片內容需使用串流服務，並支援多種畫質的自動切換
- 系統需支援與企業現有的人力資源系統 (HRIS) 進行單一登入 (SSO) 整合

請分析此線上學習平台的需求，並設計其系統架構。
```

## 6. 醫療保健：醫院資訊系統 (HIS)

### 場景說明
適用於開發或升級醫院內部管理系統，重點在於醫療流程整合、資料準確性、法規遵循與系統穩定性。

### Prompt 模板
```
# 專案：次世代醫院資訊系統 (HIS)

## 專案類型：醫院管理資訊系統

## 核心模組需求
1. 門診管理：整合線上掛號、醫生看診、電子病歷、開立處方及檢驗單的流程
2. 住院管理：涵蓋從入院登記、護理紀錄、床位管理到出院結算的完整流程
3. 藥品管理：管理藥品庫存、自動檢查處方中的藥物交互作用，並提供用藥安全警示
4. 檢驗報告：整合檢驗科設備，讓醫生能即時在系統中查看影像（如 X 光）與檢驗報告

## 關鍵要求
1. 法規遵循：系統必須完全符合醫療法規、健保規範與病患隱私保護（HIPAA）標準
2. 資料標準：需支援國際醫療資訊交換標準，如 HL7 與 DICOM
3. 系統穩定性：要求 7x24 小時不中斷服務，並具備完善的災難復原計畫
4. 資訊安全：需建立嚴格的存取控制機制，確保只有授權人員能存取病患資料

請執行完整的 AISDLC 流程，產出專業且可靠的醫療系統需求與設計方案。
```

## 7. B2B 平台：專業服務媒合

### 場景說明
適用於建立連接服務提供方與需求方的雙邊市集平台，例如設計師接案平台或企業採購平台。

### Prompt 模板
```
# 專案：B2B 設計服務媒合平台

## 核心概念
建立一個平台，讓需要設計服務的企業（需求方）可以找到並聘用專業的設計師（服務方）。

## 雙邊功能需求
1. 需求方功能：
   - 可發布設計專案需求，包含詳細說明、預算範圍與時程
   - 可搜尋、瀏覽設計師的作品集與評價
   - 可透過平台安全地支付款項（里程碑付款）
2. 服務方功能：
   - 可建立個人化的作品集頁面，展示過往案例
   - 可對感興趣的專案進行報價與提案
   - 可透過平台與客戶溝通、交付檔案並收取款項
3. 平台核心機制：
   - 建立雙向評價與評分系統
   - 提供專案進度追蹤與爭議處理機制

## 商業模式
- 向服務方收取 15% 的成交佣金
- 提供付費的專案推廣服務給需求方

請分析此 B2B 平台的需求，並特別關注其交易流程與信任機制的設計。
```

## 8. 數據分析：商業智慧 (BI) 平台

### 場景說明
適用於整合企業內外部多個資料來源，建立一個提供視覺化報表與數據洞察的商業智慧平台。

### Prompt 模板
```
# 專案：零售業銷售數據分析平台 (BI)

## 背景
公司擁有多個銷售管道（實體門市、官方網站、社群電商），數據散落在不同系統中，希望能整合分析以制定決策。

## 數據來源
1. 門市 POS 系統：每日銷售明細、會員資料
2. 官網電商資料庫：線上訂單、用戶瀏覽行為
3. 社群平台 API：廣告投放成效數據（觸及、點擊、轉換）

## 分析需求
1. 銷售分析儀表板：
   - 即時呈現各管道的銷售業績與達成率
   - 提供商品銷售排行、熱銷時段分析
2. 顧客分析儀表板：
   - 分析顧客輪廓（年齡、性別、地區）
   - 計算顧客終身價值 (LTV) 與回購率
3. 行銷活動分析：
   - 評估不同廣告活動的投資報酬率 (ROI)

## 使用者角色
- 高階主管：查看整合性的經營儀表板
- 行銷人員：深入分析廣告成效與顧客行為

請分析此數據分析平台的需求，並設計其數據倉儲架構與 ETL 流程。
```

## 9. 物聯網 (IoT)：智慧家庭 App

### 場景說明
適用於開發一款能控制和管理多種智慧家庭裝置的中央控制 App，強調裝置配對、情境模式與自動化。

### Prompt 模板
```
# 專案：智慧家庭控制中心 App

## 核心功能
1. 裝置管理：
   - 支援透過 Wi-Fi 和藍牙快速新增智慧裝置（如智慧燈泡、插座、攝影機）
   - 在 App 中即時查看所有裝置的線上狀態並進行遠端控制
2. 情境模式：
   - 允許使用者自訂情境模式，例如「回家模式」（自動開燈、開空調）、「睡眠模式」（關閉所有燈光、啟動保全）
   - 提供一鍵啟動情境的功能
3. 自動化規則：
   - 使用者可設定自動化觸發條件，例如「當溫度高於 28°C 時，自動開啟空調」或「每天早上 7 點，自動拉開窗簾」
4. 家庭共享：
   - 戶長可邀請家庭成員加入，並設定不同成員對裝置的控制權限

## 技術要求
- 需支援 Matter 智慧家庭標準，以確保跨品牌裝置的相容性
- 後端需能穩定處理大量裝置的即時狀態更新與指令傳達

請分析此智慧家庭 App 的需求，並設計其與物聯網裝置的通訊協定。
```

## 10. 客戶服務：AI 智慧客服系統

### 場景說明
適用於建立整合多通路（網站、App、社群）的智慧客服系統，利用 AI 自動回覆常見問題並輔助人工客服。

### Prompt 模板
```
# 專案：AI 智慧客服系統

## 目標
建立一個能 24/7 自動回覆客戶查詢，並在必要時無縫轉接人工客服的整合性平台。

## 核心功能
1. AI 聊天機器人 (Chatbot)：
   - 部署在官方網站與 App 中，能理解自然語言並根據知識庫回答常見問題（例如：運費、退貨政策）
   - 當 AI 無法回答時，能自動將對話轉接給在線的人工客服
2. 多通路整合：
   - 統一接收來自網站、Email、Facebook Messenger 的客戶訊息至單一後台
3. 客服工作台：
   - 人工客服可在後台查看完整的客戶對話歷史
   - 系統需提供罐頭訊息、客戶標籤等功能以提升效率
4. 知識庫管理：
   - 提供一個界面讓非技術人員也能輕鬆新增、編輯、管理 AI 回覆所依據的知識庫內容
5. 數據分析：
   - 統計客戶最常詢問的問題、AI 回答的解決率、客服處理時長等關鍵指標

請分析此智慧客服系統的需求，並建議適合的自然語言處理 (NLP) 模型架構。
```
