# 使用說明
# =============================================================================

## 如何使用此模板
1. **複製模板**：將此模板複製並重新命名（如：requirements-analysis-workflow.md）
2. **填寫基本資訊**：根據具體 workflow 需求填寫名稱、描述、適用場景
3. **定義角色責任**：明確主要負責人、參與者和審查者
4. **設計執行步驟**：詳細規劃每個步驟的作業內容和檢查點
5. **確定交付標準**：明確輸出物和驗收條件
6. **建立協作關係**：定義與其他 workflows 和 agents 的關聯
7. **移除註釋**：清理所有指導性註釋，保持文檔簡潔

## 最佳實踐
- **具體化**：避免抽象描述，提供具體可執行的步驟
- **可測量**：所有標準和條件都應該是可檢驗的
- **角色明確**：每個步驟都要有明確的執行者和責任人
- **品質導向**：建立充分的檢查點和品質標準
- **協作友好**：考慮與其他 workflows 和 agents 的整合

# AISDLC 整合說明

## 與 Agent 配置整合
此模板設計為與 AISDLC 的 Agent 配置完全相容：
- workflows 可直接對應 agent 配置中的 `supported_workflows`
- 角色責任對應 agent 的 `collaboration_rules`
- 交付物對應 agent 的 `document_responsibilities`

## 文檔流程整合
workflows 應支援 AISDLC 的核心文檔流程：
```
PRD → FRD → SRD → Implementation → AT → Test Report
```

## 追蹤鏈支援
確保 workflow 支援完整的需求追蹤鏈：
```
Business Need → User Story → Acceptance Criteria → Acceptance Test → Implementation
```
